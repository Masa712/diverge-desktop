# Diverge Desktop — 要件定義書

**バージョン**: 1.0
**作成日**: 2026-04-06
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 プロジェクト名
**Diverge Desktop**（仮称）

### 1.2 概要
Diverge AI（Webアプリ）のコアコンセプトである「ノードベースの分岐型AIチャット」をmacOSデスクトップアプリとして再実装する。オープンソースLLMをローカル実行することで、**インターネット接続不要・完全プライベート・無料**で利用できるAIチャット環境を提供する。

### 1.3 目的・背景

| 課題 | 解決策 |
|------|--------|
| クラウドLLM APIのコスト | ローカルLLM実行（無料） |
| プライバシー懸念（会話データがクラウドに送信される） | 完全オフライン処理 |
| インターネット接続依存 | オフライン動作 |
| LM Studioなど既存ツールのUI/UX限界 | DivergeのノードベースUIの優位性 |

### 1.4 スコープ

**対象プラットフォーム**: macOS 13 Ventura 以降（Apple Silicon / Intel 両対応）
**対象ユーザー**: 個人開発者、研究者、プライバシー重視のユーザー

---

## 2. ターゲットユーザー

### 2.1 プライマリユーザー

**ローカルAI活用者**
- LM Studio / Ollama などを使っているが、UIに不満がある
- 複数の会話パスを試したい（プロンプトエンジニアリング）
- Apple Silicon MacでLLMを高速実行したい

### 2.2 セカンダリユーザー

**Diverge Webユーザー**
- 既存のDiverge AIユーザーがローカル版に移行または併用

## 3. 機能要件

### 3.1 LLMエンジン管理

#### FR-01: モデルダウンロード・管理
- [ ] Ollamaが提供するモデルライブラリからモデルを検索・ダウンロード

- [ ] Apple Siliconのメモリ設定（コンテキスト長、GPU レイヤー数）

- [ ] Temperature（0.0〜2.0）

#### FR-06: チャット履歴管理

## 4. 非機能要件

### 4.1 パフォーマンス

| 指標 | 目標値 |
|------|--------|
| アプリ起動時間 | 3秒以内 |
| UI操作のレスポンス | 100ms以内 |
| メモリ使用量（アプリのみ、LLM除く） | 200MB以下 |
| 初回トークン出力までの時間 | モデルロード後 1秒以内 |

### 4.2 セキュリティ・プライバシー

- 会話データはローカルのみに保存（クラウドに一切送信しない）
- APIキー等の機密情報はmacOSキーチェーンに保存
- ネットワーク通信はユーザーが明示的に有効化した場合のみ（OpenRouter連携時など）
- アプリ自体のアップデート確認は任意設定

### 4.3 ユーザビリティ

- macOSのHuman Interface Guidelines（HIG）に準拠
- VoiceOverアクセシビリティ対応（基本レベル）
- キーボードのみで主要操作が完結すること
- 初回起動時のオンボーディングフロー（Ollama未インストール検出→インストールガイド）

### 4.4 互換性

| 項目 | 要件 |
|------|------|
| OS | macOS 13 Ventura 以降 |
| アーキテクチャ | Apple Silicon (M1/M2/M3/M4) および Intel |
| Ollama | v0.3.0 以降 |
| 配布形式 | Universal Binary (.dmg) |

### 4.5 信頼性

- LLMクラッシュ時の自動再起動
- アプリクラッシュ時のセッション自動保存（5分ごと + 変更時）
- Ollamaサーバー切断時の自動再接続

## 5. 制約条件

### 5.1 技術的制約
- Ollamaがシステムにインストールされていることが前提（初回セットアップで案内）
- 大型モデル（70B以上）はApple Silicon 64GB以上のメモリが必要
- Intel MacではGPUアクセラレーションが制限される

### 5.2 ビジネス的制約
- 初期リリースはmacOSのみ（Windows/Linux対応は将来検討）
- App Store配布はOllama連携のサンドボックス制限により困難 → 直接配布（.dmg）を優先

## 6. リリース計画

### Phase 1 — MVP（最小実用製品）
- Ollamaを介したローカルLLM実行
- 基本チャット機能（ノードベース、ストリーミング）
- セッション保存・復元
- モデル管理（ダウンロード・削除・切替）

### Phase 2 — 機能拡充
- ファイル・画像アタッチ
- OpenRouter連携（クラウドモデル混在）
- 推論パラメータの詳細設定
- セッション検索・エクスポート

### Phase 3 — 高度な機能
- RAG（ローカルドキュメントを使った検索拡張生成）
- モデルの微調整（Fine-tuning）サポート
- プラグイン機能

## 7. 用語定義

| 用語 | 定義 |
|------|------|
| ノード | チャットの1メッセージ（AIまたはユーザー）を表す単位 |
| ブランチ | 既存ノードから分岐した新しい会話パス |
| セッション | ノードツリー全体を含む1つの会話単位 |
| GGUF | llama.cppが使用するモデルファイルフォーマット |
| Ollama | ローカルLLM実行のためのオープンソースツール |
| Metal | Apple製GPUアクセラレーションフレームワーク（macOS） |
